# Plan 1 — Design Rules and Foundations

## Goal

Establish the Mini Rally design language and token architecture in Figma before creating UI components.

## Inputs

- Confirmed Plan 0 artifacts and decisions.
- Current mockup CSS/font evidence.
- Target Figma file and its preflight result.

## Checklist

- [x] P1.a Inspect target Figma file/pages/libraries; post the Phase 1 checklist before the first mutation.
- [x] P1.b Create/confirm file information architecture: Cover, Getting Started, Foundations, separators, Components, Utilities.
- [x] P1.c Write project design rules: naming, auto-layout, responsive behavior, accessibility, icon and asset policy.
- [x] P1.d Build primitive collections for color, spacing, radius and typography values.
- [x] P1.e Build semantic variables; alias primitives; set variable scopes and CSS code syntax.
- [x] P1.f Create text and effect styles; verify Inter font family and exact style names.
- [x] P1.g Build Foundations documentation pages with colour/type/spacing/radius/elevation examples.
- [x] P1.h Validate variable counts, scopes, aliases, style names, screenshots and naming.
- [x] P1.i Write `P1_DESIGN_RULES.md` and `P1_TOKEN_MAPPING.md` in this folder.

## Deliverables

- Native Figma Foundations pages and Variables/Styles.
- `P1_DESIGN_RULES.md`, `P1_TOKEN_MAPPING.md`, `P1_VALIDATION.md`.

## Exit criteria

- Components can bind all core visual properties to tokens.
- No variable has `ALL_SCOPES`; semantic values alias primitives.
- CSS/Dev Mode syntax maps to approved names.
- Foundations are visually reviewed and font is asserted.

Set tracker to `AWAITING PLAN 1 CONFIRMATION`; stop until `CONFIRM PLAN 1`.
