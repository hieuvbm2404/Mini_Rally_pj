# P3-TS-01 - Team Status

## Scope

Verify Iteration-scoped Task grouping, Task identity, state controls, time values and parent Story/Defect roll-up.

## Approved baseline

- Tasks belong to the selected Iteration through their parent US/DE.
- Task State is exactly `Defined`, `In-Progress`, `Completed`.
- When all child Tasks are Completed, the parent becomes `Completed`.
- Reopening a completed Task moves the parent to `In-Progress`, including when the parent was manually promoted to `Accepted`.
- Missing Task time values display as zero.
- The approved route and breadcrumb are `[Project] > Track > Team Status`.

## Recorded DevInt execution

- Parent auto-completion and reopen-to-`In-Progress` were confirmed on `US-8 / TA-6`.
- Reopening a Task under Accepted parent `US-11 / TA-8` left the parent at `Accepted`.
- Team Status exposed extra local Search, Filters, Show Fields and pagination.
- Breadcrumb rendered `ACME Corp > Team Status`.
- State was rendered as an icon toggle and some labels were not visible.
- Missing Estimate, To Do and Actuals values rendered as dashes.
- State wording included `Define`, `In Progress` and `Complete`.

## Result and gaps

Result: **Partial**.

Confirmed findings: `GAP-P3-TS-001` through `GAP-P3-TS-007`.

The BA-approved fix for `GAP-P3-TS-004` is that reopening a child Task forces the parent back to `In-Progress` even if the parent is currently `Accepted`.
