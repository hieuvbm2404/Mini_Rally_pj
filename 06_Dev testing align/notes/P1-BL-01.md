## Decisions recorded on 2026-07-19

### Audit correction: filter execution is working

- The first automation pass used `fill("")` to clear a no-result search. The visible value did not clear, so every following filter was combined with the stale no-match query.
- Retest with the user-equivalent sequence Control+A and Backspace restored both mockup and DevInt.
- Type Story, Schedule State Accepted, Owner Admin User, Release v2.0 and Iteration Sprint 26.1 then returned the expected records.
- Classification: invalidated finding, not a DevInt defect.

### 2. Missing Priority filter - fix approved

- Mockup Manage Filters includes Priority.
- SRS requires Defect Priority filter values Low/Normal/High/Urgent/None.
- DevInt displays Priority as a column but has no Priority filter.
- BA decision: add the SRS Priority filter to DevInt.

### 3. Pagination total changes with page size - defect confirmed

- At page size 10: Page 1 showed 1-10 and Page 2 showed 11-19.
- At page size 25 on the same clean context: the footer showed only 1-12 and Next was disabled.
- The inconsistency reproduced twice.
- BA decision: record this as a DevInt defect for the development team.
- Fix proposal: fix API total/ID membership or FE request/cache behavior; assert the same total and item IDs across page sizes.

### 4. Filter configuration model differs - DevInt accepted

- Mockup uses Manage Filters, lets the user choose filter fields, renders removable filter cards and provides Clear filters.
- DevInt uses a fixed panel containing Type, Schedule State, Owner, Release and Iteration.
- BA decision: keep the DevInt fixed panel for the current delivery and add Priority.
- Reconciliation: the mockup/document wording may be adjusted to the accepted DevInt baseline.
- Future Backlog: Manage Filters and other configurable/additional filter fields.

### 5. Column sorting - pass

- DevInt sortable headers are acceptable.
- Type sorting was verified without mutating Work Item rank.

## Confirmed: Flow State and Schedule State

- Confirmed catalog: Idea, Defined, In-Progress, Completed, Accepted, Release.
- Correct presentation requested by BA: Flow State is a six-box segmented control; Schedule State is a dropdown.
- DevInt currently places the six-box segmented control under the `Schedule State` label. This is the Flow State control, while the actual Schedule State dropdown is missing.
- The current mockup presents Schedule State as a dropdown but does not yet show the separate Flow State six-box control on the Backlog list.
- BA confirmed the target on 2026-07-19.

### Dev action

1. Relabel/reclassify the current six-box segmented control as `Flow State`.
2. Add the separate `Schedule State` field as a dropdown.
3. Use the exact values `Idea`, `Defined`, `In-Progress`, `Completed`, `Accepted`, `Release` for both fields.
4. Default both fields to `Idea` when a Work Item is created.
5. Synchronize the two values: changing either field immediately updates the other.
6. When all child Tasks are Completed, automatically set the parent US/DE to `Completed`.
7. When any completed child Task is reopened, automatically move the parent US/DE back to `In-Progress` and recalculate metrics.
8. Keep manual status editing available; automation is a convenience and does not lock user changes.

### BA mockup/SRS action

1. Keep `Schedule State` as a dropdown.
2. Add the separate `Flow State` six-box control to the Backlog list.
3. Document the exact six-state catalog, default `Idea`, two-way synchronization, Task-completion automation and manual override behavior.
