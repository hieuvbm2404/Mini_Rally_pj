## BA decisions confirmed on 2026-07-19

### 1. Every editable field must save inline

- BA rule: every field exposed as editable in a Backlog row must autosave inline. Clicking out must retain and display the saved value; the user must not have to open Detail or reload.
- Direct reproduction on US-1:
  1. Click Title to open the inline textbox.
  2. Change `Upgrade NX workspace to v21` to `Upgrade NX workspace to v21 [AUDIT]`.
  3. Click the Backlog search field.
  4. The row immediately displayed the old title, appearing not to save.
  5. Reload displayed the new `[AUDIT]` title, proving the backend mutation succeeded while the row UI/cache stayed stale.
  6. The original title was restored and verified after reload.
- Dev action: on mutation success, update the row from the canonical response or invalidate/refetch the correct scoped Backlog query. Keep the new value on success; show an explicit error and rollback only on failure. Apply the same loading/save/error contract to Title and every other inline-editable field.

### 2. Keep DevInt drag rank

- Mockup: visible Move up/Move down buttons.
- DevInt: hover-only drag handle; drag and persistence work.
- BA decision: retain DevInt drag as the primary interaction and update mockup/SRS to match.
- Dev action: verify keyboard reordering, visible focus and screen-reader naming. Add visible up/down actions only if the keyboard contract is insufficient.
